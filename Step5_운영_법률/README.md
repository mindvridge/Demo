# Step 5: 운영 법률

## 📍 이 폴더의 목적

TarotMind의 **법률 리스크 관리, 규제 준수, 개인정보보호** 전략을 다룹니다.
서비스 출시 전 반드시 검토해야 할 법률적 이슈와 대응 방안이 포함되어 있습니다.

---

## 📄 문서 목록

### 01_법률_검토_착수_계획서.md ⭐⭐⭐⭐
**읽기 시간**: 2시간
**대상**: 법무팀, 컴플라이언스, CTO, 경영진

**주요 내용**:

1. **점술 관련 규제**
   - **문제**: 경범죄처벌법 제1조 제30호 (점술 영업)
   - **대응**:
     - "심리분석 도구"로 포지셔닝
     - 명확한 면책조항
     - "엔터테인먼트" 목적 명시
   - **예시 문구**:
     > "본 앱은 타로 카드를 활용한 심리 분석 및 자기성찰 도구입니다. 미래 예측이나 운명 결정의 목적이 아닙니다."

2. **개인정보보호법**
   - **요구사항**:
     - 개인정보 수집 동의
     - 암호화 (AES-256)
     - 최소 수집 원칙
   - **대응**:
     - PIMS (개인정보보호 관리체계) 인증
     - 개인정보처리방침 작성
     - 동의 관리 시스템
     - 정기 보안 감사

3. **전자상거래법**
   - **요구사항**:
     - 사업자 정보 공개
     - 환불 정책 명시
     - 청약철회권 (7일)
   - **대응**:
     - 명확한 이용약관
     - 결제 전 최종 확인 화면
     - 환불 프로세스 자동화

4. **전자금융거래법**
   - **요구사항**:
     - 금융보안원 보안성 심사
     - PCI DSS 준수 (카드 결제)
   - **대응**:
     - 토큰화 (Tokenization)
     - 결제 정보 저장 금지
     - PG사 연동 (토스페이먼츠, 이니시스)

5. **저작권법**
   - **요구사항**:
     - 타로 카드 이미지 저작권
     - AI 생성 이미지 저작권
   - **대응**:
     - Midjourney 상용 라이선스 구매
     - 자체 제작 이미지 사용
     - 오픈소스 타로덱 활용

6. **GDPR (글로벌 확장 시)**
   - **요구사항** (EU 사용자):
     - 동의 관리
     - 데이터 이동권
     - 삭제권 ("잊힐 권리")
   - **대응**:
     - EU 지역 서버 (AWS eu-west-1)
     - GDPR 준수 동의 화면
     - 데이터 내보내기 기능
     - 계정 삭제 즉시 처리

7. **의료법 (심리 상담 관련)**
   - **문제**: 심리 상담 서비스로 오인될 위험
   - **대응**:
     - "의료 서비스 아님" 명시
     - 심리 상담사 연결 서비스 제외
     - 위기 상황 시 전문 기관 안내 (1393 자살예방센터 등)
